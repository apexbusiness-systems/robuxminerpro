# Sprint Cadence

## Project Board

Board URL: [Project Board](https://github.com/your-org/your-repo/projects/1)

## Kanban Columns

**Backlog** → **Ready** → **In Progress** → **In Review** → **Done**

## WIP Limits (Advisory)

WIP limits are **advisory** - aim for focus and flow, but exceptions allowed by team agreement.

*Rationale*: [Kanban WIP limits](https://www.atlassian.com/agile/kanban/wip-limits) help maintain sustainable pace and identify bottlenecks while allowing flexibility for urgent work.

## Sprint Rhythm

- **Sprint Length**: 2 weeks
- **Planning**: Sprint start (define Ready items)
- **Review**: Sprint end (demo Done items)
- **Retrospective**: Continuous improvement focus

## Definition of Done

- [ ] Code reviewed and approved
- [ ] Tests passing (where applicable)
- [ ] Documentation updated
- [ ] UX/accessibility validated
- [ ] Ready for release tagging

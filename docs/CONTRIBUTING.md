# RobuxMinerPro — Contributing Guide

> **Mandatory reading for all agents and developers.** Violations of these rules cause regressions.

---

## Commit Convention

All commits MUST follow [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): description

feat(auth):     new feature in auth module
fix(ui):        bug fix in UI
docs(arch):     documentation update
refactor(nav):  code restructuring
chore(deps):    dependency update
test(e2e):      test addition/fix
perf(dashboard): performance improvement
```

**Rules:**

- Scope is required (use the component/module name)
- Description is lowercase, no period, imperative mood
- Max 72 characters for subject line
- Body is optional but recommended for non-trivial changes

---

## Quality Gates (MANDATORY)

Every change MUST pass ALL gates before push:

```bash
npx tsc --noEmit       # TypeScript — zero errors
npx eslint .           # ESLint — zero errors (warnings acceptable)
npm run build          # Vite production build — exit 0
```

**CI will reject** any push that fails these gates.

## Anti-Drift Rules

### NEVER Modify (without explicit approval)

| File/Pattern                                         | Reason                                    |
| ---------------------------------------------------- | ----------------------------------------- |
| `App.tsx` provider order                             | Breaks context dependencies               |
| `src/integrations/supabase/types.ts`                 | Auto-generated by Supabase CLI            |
| `components/ui/*` (48 files)                         | shadcn/ui primitives — upstream managed   |
| Hero container: `max-w-[1100px]`, `gap-12 lg:gap-20` | Calibrated layout constraints             |
| `ThemeProvider` storage key `rmp-theme`              | Breaking change affects all users         |
| `signOut()` fire-and-forget pattern                  | Reverting to `await` causes sign-out hang |

### NEVER Do

| Action                                                     | Consequence                             |
| ---------------------------------------------------------- | --------------------------------------- |
| Import large binaries via Vite module system               | Crashes `vite-plugin-pwa` build         |
| Hardcode user-facing strings (skip `t()`)                  | Breaks internationalization             |
| Hardcode light/dark colors (skip `dark:` prefix)           | Breaks theme switching                  |
| Use `localStorage.clear()` in auth code                    | Nukes theme prefs, PWA state, analytics |
| Show dev/mock features without `import.meta.env.DEV` guard | Leaks to production                     |
| Redirect sign-out to `/` instead of `/auth`                | User appears still logged in            |
| Add `await` to `supabase.auth.signOut()` in `signOut()`    | Causes 30s+ hang on network issues      |

### ALWAYS Do

| Action                                                     | Reason                       |
| ---------------------------------------------------------- | ---------------------------- |
| Run `npm run build` before pushing                         | Catches CI failures locally  |
| Use `public/` for media >1MB                               | Prevents bundler crashes     |
| Gate dev features behind `import.meta.env.DEV`             | Tree-shaken from prod builds |
| Use `t('key')` for all user-facing text                    | i18n compliance              |
| Use feature flags for experimental nav items               | Prevents accidental exposure |
| Clear `sb-*` keys (not `localStorage.clear()`) on sign-out | Preserves user preferences   |

## File Ownership

| Owner         | Files                                                            | Approval Required           |
| ------------- | ---------------------------------------------------------------- | --------------------------- |
| Architecture  | `App.tsx`, `main.tsx`, provider order                            | Always                      |
| Auth          | `useAuth.tsx`, `AuthPage.tsx`, `ProtectedRoute.tsx`, `client.ts` | Always                      |
| Navigation    | `Navigation.tsx` sign-out wiring                                 | Always                      |
| Database      | `supabase/types.ts`, `supabase/migrations/*`                     | Always                      |
| UI Primitives | `components/ui/*`                                                | Upstream update only        |
| Pages         | `pages/*.tsx`                                                    | Standard review             |
| Styles        | `index.css`, `tailwind.config.ts`                                | Design review               |
| CI/CD         | `.github/workflows/*`, `vercel.json`                             | Ops review                  |
| Docs          | `docs/*`                                                         | Self-service (keep current) |

---

## PR Requirements

1. **Title:** Follows commit convention (`type(scope): description`)
2. **Description:** What changed, why, and how to verify
3. **Gates passed:** tsc, eslint, build (mentioned in PR body)
4. **No scope creep:** One logical change per PR
5. **Docs updated:** If architecture, UI, or ops behavior changed

## Environment Setup

```bash
# 1. Clone
git clone https://github.com/apexbusiness-systems/robuxminerpro.git
cd robuxminerpro

# 2. Install
npm install

# 3. Environment
cp .env.example .env
# Edit .env with real Supabase credentials

# 4. Dev server
npm run dev
# Open http://localhost:8080

# 5. Verify
npx tsc --noEmit && npx eslint . && npm run build
```

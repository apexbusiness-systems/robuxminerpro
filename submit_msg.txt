🔒 Add Authorization header to leads endpoint request

🎯 **What:** The LeadCaptureModal makes an unauthenticated POST request to the external leads endpoint configured via VITE_LEADS_ENDPOINT_URL. This change introduces a new environment variable `VITE_LEADS_API_KEY` and passes it as a Bearer token in the Authorization header.

⚠️ **Risk:** Without authentication, the leads endpoint could be subjected to unauthorized access, spam submissions, or abuse, potentially leading to data pollution or denial of service against the leads system.

🛡️ **Solution:** Modified `src/components/LeadCaptureModal.tsx` to conditionally read the API key from `import.meta.env.VITE_LEADS_API_KEY` and inject it into the `Authorization: Bearer <token>` header if present. Also updated `.env.example` and `.env.ci` to document the new expected environment variable.

## A: Position to Win

**Input**: Product, market, competitors  
**Output**: Messaging that converts  
**Cost**: $0 | **Automation**: Template-driven

### Universal Positioning Formula
```
For [TARGET] who [PAIN],
[PRODUCT] is the [CATEGORY] that [UNIQUE MECHANISM],
unlike [ALTERNATIVE] which [THEIR WEAKNESS].
```

### The 10-Second Test
If a stranger can't repeat your value prop after hearing it once, it fails.

### Messaging Hierarchy (Create All 4)
```
L1 TAGLINE (5 words): [Outcome] + [Differentiator]
L2 HEADLINE (12 words): [Pain] → [Solution] → [Proof]
L3 ELEVATOR (30 sec): Problem → Agitate → Solution → Proof → CTA
L4 NARRATIVE (2 min): Origin → Discovery → Solution → Results → Vision
```

## C: Close More Deals

**Input**: Lead source, objections, sales cycle  
**Output**: Repeatable sales system  
**Success**: Close rate > industry benchmark + 20%

### Universal Discovery Framework
```
SITUATION (2 min): "Walk me through how you handle [X] today."
PROBLEM (3 min): "What breaks? What's frustrating?"
IMPACT (2 min): "What does that cost you—time, money, stress?"
PRIORITY (1 min): "Where does solving this rank for you?"
TIMELINE (1 min): "What's driving the timeline?"
BUDGET (1 min): "What have you allocated for this?"
```

### Objection Response Engine
| Objection | Type | Response |
|-----------|------|----------|
| "Too expensive" | Price | "Compared to what? [Pause] What's the cost of NOT solving this?" |
| "Need to think" | Stall | "Of course. What specifically do you want to think through?" |
| "Not now" | Timing | "Understood. What would need to change for this to be a priority?" |
| "Send info" | Brush-off | "Happy to. What specific questions should it answer?" |
| "Need approval" | Authority | "Makes sense. What would [decision-maker] need to see to say yes?" |

### Close Acceleration Tactics
```
URGENCY (ethical):
- "This pricing is locked until [date]"
- "Implementation slots fill 2 weeks out"
- "Pilot program limited to [number] companies"

RISK REVERSAL:
- "30-day pilot, cancel anytime"
- "Money-back guarantee"
- "Start with smallest package"

SOCIAL PROOF:
- "[Similar company] saw [result] in [timeframe]"
- "We have [X] customers in your space"
- [Share relevant case study]
```

### Sales Automation
```

- Meeting scheduling (Calendly/SavvyCal)
- Proposal generation (PandaDoc/Qwilr)
- Follow-up sequences (CRM workflows)
- Contract signing (DocuSign/HelloSign)
- Onboarding kickoff (automated email)

DON'T AUTOMATE:
- Discovery calls
- Objection handling
- Negotiation
- Relationship building
```

## E: Optimize What's Working

**Input**: Current metrics  
**Output**: Prioritized improvement plan  
**Success**: 20%+ improvement in primary metric within 30 days

### Metric Priority Stack
```
FIX IN THIS ORDER:
1. CONVERSION RATE (free traffic is useless if it doesn't convert)
2. AVERAGE ORDER VALUE / DEAL SIZE (more revenue per customer)
3. RETENTION / REPEAT RATE (keep what you've earned)
4. TRAFFIC / LEAD VOLUME (scale what's working)
5. CAC (optimize efficiency last, not first)
```

### Quick Diagnostic
```
SYMPTOM → LIKELY CAUSE → FIX

High traffic, low leads → Bad offer or CTA → Test new lead magnet
High leads, low sales → Lead quality or sales process → Tighten ICP, fix discovery
High CAC → Wrong channel or weak creative → Test new channels, refresh creative
High churn → Onboarding or product-market fit → Fix first 30 days, talk to churned users
Low traffic → SEO, distribution, or awareness → Content + distribution focus
```

### A/B Test Prioritization (ICE)
```
IMPACT (1-10): How much will this move the metric?
CONFIDENCE (1-10): How sure are we it'll work?
EASE (1-10): How fast can we ship it?

SCORE = (I + C + E) ÷ 3

Test highest scores first. One test at a time.
```

### IMMEDIATE ACTIONS
1. Identify your #1 broken metric
2. Diagnose using table above
3. Design one test (highest ICE score)
4. Run for 7-14 days with statistical significance
5. Winner becomes new control; repeat

## G: Price for Profit

**Input**: Product, market, costs, competitors  
**Output**: Pricing strategy + validation plan  
**Success**: Price captures maximum value without killing conversion

### Pricing Model Selection
| Model | Best When | Example |
|-------|-----------|---------|
| Flat rate | Simple product, easy comparison | "$99/month" |
| Per seat | Value scales with team size | "$10/user/month" |
| Usage-based | Value scales with consumption | "$0.01/API call" |
| Tiered | Different segments, different needs | Good/Better/Best |
| Freemium | Viral potential, low marginal cost | Free tier + paid upgrade |

### Price Setting Framework
```
1. COST FLOOR: What does it cost you to deliver? (Never go below)
2. VALUE CEILING: What's it worth to the customer? (Theoretical max)
3. COMPETITOR ANCHOR: What do alternatives cost?
4. YOUR PRICE: Between floor and ceiling, relative to anchor

FORMULA:
Price = (Value Ceiling × 0.1 to 0.3) 
→ Capture 10-30% of value created
→ Rest goes to customer as "surplus" (why they buy)
```

### Pricing Psychology
```
ANCHORING: Show highest tier first
DECOY: Middle option should be "obvious" choice
CHARM: $99 beats $100 (left-digit effect)
BUNDLING: Combine for perceived value
ANNUAL: Offer discount for commitment (10-20%)
```

### Price Validation Test
```
1. Survey 20 prospects: "What would you expect to pay?"
2. Test 2 prices with real landing pages (A/B)
3. Measure: Conversion rate × Price = Revenue per visitor
4. Winner = highest revenue per visitor, not highest conversion
```

### IMMEDIATE ACTIONS
1. Calculate your cost floor
2. Estimate value ceiling (ask 5 customers)
3. Research 3 competitor prices
4. Set initial price using framework
5. Test within 2 weeks

## H: Scale with Growth Loops

**Input**: Product, user behavior, growth goals  
**Output**: Self-sustaining growth engine  
**Success**: Organic growth > paid growth

### Growth Loop Architecture
```
USER ACTION → EXTERNAL EXPOSURE → NEW USER → USER ACTION
     ↑                                           │
     └───────────────────────────────────────────┘
     
Example (Referral):
User invites friend → Friend sees value → Friend signs up → Friend invites friend
```

### Loop Type Selection
| Product Type | Best Loop | Example |
|--------------|-----------|---------|
| Viral product (network effects) | Inherent virality | Slack, Dropbox |
| Content product | Content loop | HubSpot, Ahrefs |
| Community product | Community loop | Reddit, Discord |
| Any product | Referral program | Any |

### Referral Program Blueprint
```
STRUCTURE:
- Give: [What referrer gets]
- Get: [What new user gets]
- Trigger: [When to ask]
- Mechanic: [How to share]

BEST PRACTICES:
- Ask after value moment (not signup)
- Make sharing 1-click
- Show progress (referred 2 of 5)
- Reward both sides
- Test incentive amount

AUTOMATION:
- Referral tracking: ReferralCandy, Rewardful, FirstPromoter
- Or build simple: unique referral codes + tracking
```

### Viral Coefficient Math
```
K = invites_per_user × conversion_rate

K > 1 = True viral growth (rare)
K = 0.5 = 50% growth boost (good)
K = 0.3 = 30% growth boost (normal)

IMPROVE K BY:
- Increasing invite prompts
- Making invites easier
- Improving invite conversion
- Incentivizing more invites
```

### IMMEDIATE ACTIONS
1. Identify your natural sharing moment
2. Design referral program using blueprint
3. Implement with free/cheap tool
4. Measure K-factor after 30 days
5. Optimize invite flow
